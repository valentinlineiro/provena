# v0.4.0 — Adoption

**Date:** 2026-07-23

## From architecture to adoption

This version shifts the focus from proving the architecture to proving that
someone can use it. A developer who discovers Provena today can go from
`git clone` to `provena render my-profile` in under five minutes, without
reading architectural documentation.

## What changed

### CLI v2

- `provena render <workspace>` — default format (markdown)
- `provena render <workspace> --format jsonresume` — JSON Resume output
- `provena validate <workspace>` — load + validate, exit 0/1
- `--stdout` — pipe to other tools
- `--help` — per-command help with examples
- Error messages with actionable hints

### Documentation

- README rewritten: Quick Start first, then Golden Path, then API table
- Every command in the README is executable verbatim — no footnotes

### Website

- Landing page updated with `identity → resume.md + resume.json` demo
- New Quick Start page (try it in 5 minutes)
- Status table showing what's shipped
- Architecture page updated with current contracts (Projector, Renderer)

### Golden Path

A clean checkout SHALL satisfy:

```bash
git clone <repo>
npm install
npm run build
provena render examples/valen
provena render examples/valen --format jsonresume
provena validate examples/valen
```

This is the executable specification of the adoption experience.
